- Arquivo diário criado para evitar falha de leitura no bootstrap.
- Hoje configuramos o WhatsApp/OpenClaw para operar com prefixo `/mestre` e persona Yoda mais leve/natural (80% humor, 20% sarcasmo), com `responsePrefix` simplificado para `Yoda:`.
- Grupos allowlisted por JID configurados até agora: `120363406042795123@g.us`, `120363139629623599@g.us`, `120363424359612451@g.us`.
- Aprendizado importante: em `channels.whatsapp`, `groups` é allowlist de grupos/JIDs; `groupAllowFrom` é allowlist de remetentes dentro dos grupos. Antes houve confusão entre os dois.
- Outro aprendizado importante: `/mestre` não conta como menção nativa do WhatsApp; `requireMention: true` bloqueia esse fluxo. Para grupos via `/mestre`, manter `requireMention: false`.
- Houve falha real no bootstrap porque `memory/2026-04-01.md` não existia; criar esse arquivo destravou respostas do agente.
- Houve config inválida ao tentar usar `channels.whatsapp.dms.requireMention: true` (schema esperava objeto, não boolean). Isso contribuiu para comportamento instável/turnos terminados.
- Removidas referências stale ao plugin `oxylabs-ai-studio-openclaw` do `openclaw.json` para reduzir warnings de `plugin not found`.
- Questão ainda em aberto: hard-gate absoluto de "só responder com /mestre" não ficou garantido apenas por config/prefixo; comportamento observado indica que o pipeline do WhatsApp pode ignorar `messagePrefix` como bloqueio duro em alguns casos. Precisará investigação/implementação mais funda.
